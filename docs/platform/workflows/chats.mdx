Chat agents are multi-turn conversational agents. Each chat pairs a scenario's tools with a model and is served over the [A2A protocol](https://github.com/google/a2a). You can chat with them directly on the platform or connect them to other systems via the A2A endpoint.

A scenario qualifies as a chat when it declares `chat=True` or its first argument is an array type (conversation history).

## How to Use

### From the Platform

1. Go to **Agents → Chats**
2. Click **New** to create a chat agent
3. Select a chat scenario and model
4. Chat directly in the browser — each message produces a traced result

### From the Environments Page

1. Open any environment and load a chat scenario
2. The "Make Chat Agent" section lets you jump to the agents page

## Building a Chat Scenario

Use `chat=True` on the scenario decorator:

```python
from hud import Environment

env = Environment(name="my-env")

@env.scenario("assistant", chat=True)
async def assistant(
    messages: list[dict] | None = None,
    system_prompt: str = "You are a helpful assistant.",
) -> Any:
    """Multi-turn conversation with tools."""
    prompt_messages = [system_prompt]
    if messages:
        prompt_messages.extend(messages)

    response = yield prompt_messages
    yield 1.0 if response else 0.0
```

The `messages` array contains the conversation history. The platform manages this automatically — each `chat.send()` call appends to the history and passes the full context.

## A2A Integration

Every chat agent exposes an A2A endpoint:

```
GET  /agents/{agent_id}/.well-known/agent-card.json
POST /agents/{agent_id}/chat
```

You can connect chat agents to other A2A-compatible systems, use them as sub-agents, or serve them as standalone endpoints.

## SDK Usage

```python
from hud import Environment
from hud.services import Chat

env = Environment("my-env")
chat = env.chat("assistant", model="claude-sonnet-4-6")

r1 = await chat.send("Hello!")
r2 = await chat.send("Tell me more about that.")

# Serve as A2A endpoint
chat.serve(port=9999)
```
