The `hud init` command scaffolds a new HUD environment package. It is purely local: no network, no API key, no prompts.

## Usage

```bash
hud init NAME [OPTIONS]
```

## Arguments

<ParamField path="name" type="string" required>
  Environment name — the directory to create.
</ParamField>

## Options

<ParamField path="--dir" type="string" default=".">
  Parent directory where the package will be created. Short: `-d`
</ParamField>

<ParamField path="--force" type="boolean" default="false">
  Overwrite existing files if the directory is not empty. Short: `-f`
</ParamField>

## What It Creates

```
my-env/
├── env.py            # Environment: capabilities + @env.task tasks
├── tasks.py          # The Tasks to evaluate (hud eval tasks.py <agent>)
├── Dockerfile.hud    # Container config for deployment
└── pyproject.toml    # Dependencies and metadata
```

## Examples

```bash
# Create ./my-env
hud init my-env

# Create ./envs/my-env
hud init my-env --dir envs

# Overwrite an existing non-empty directory
hud init my-env --force
```

## Next Steps

<Steps>
<Step title="Define tasks">
Edit `env.py` — a `@env.task` is an async generator: it yields a prompt, then (after the agent answers) yields a reward.
</Step>

<Step title="Run an agent">
```bash
hud eval tasks.py claude
```
</Step>

<Step title="Deploy">
```bash
hud deploy             # Build remotely & deploy to platform
```
</Step>
</Steps>
