You've built an environment, written a scenario, and run `hud eval` locally. Now run the same tasks at scale on HUD infrastructure — hundreds of parallel runs, no local compute.

## Prerequisites

You need:
- A working environment (`env.py` with tools and scenarios)
- Tasks that pass locally (`hud eval tasks.py claude`)
- `HUD_API_KEY` set (`hud set HUD_API_KEY=your-key`)

## Step 1: Deploy

Push your environment to the platform:

```bash
hud deploy
```

This builds a Docker image remotely and registers your environment. Your scenarios become available for remote runs. Takes 2-5 minutes the first time; subsequent deploys are faster due to layer caching.

Alternatively, connect a GitHub repo on the platform for automatic rebuilds on push — see [Hosted Running](/quick-links/deploy) for GitHub integration and environment variables.

<Tip>
If your environment is pure Python (no system deps), this step is still required for remote execution — the platform needs a container image to spin up isolated instances.
</Tip>

## Step 2: Sync Tasks

Push your local task definitions to a platform taskset:

```bash
hud sync tasks my-taskset
```

This creates a taskset called "my-taskset" on the platform, uploads your tasks, and stores the taskset ID locally in `.hud/config.json`. On subsequent runs, `hud sync tasks` re-syncs to the same taskset.

The sync is diff-aware — it only uploads tasks that changed (by comparing scenario args, validation, columns, and agent_config).

## Step 3: Run Remotely

```bash
hud eval my-taskset claude --remote --full
```

Both the agent and environment run on HUD infrastructure. No local compute, no local Docker. Results stream to the platform in real-time.

| Flag | What it does |
|------|-------------|
| `--remote` | Run on HUD infrastructure instead of locally |
| `--full` | Run all tasks (without this, only the first task runs) |
| `--group-size 3` | Run each task 3 times (for variance estimation) |

Monitor progress at [hud.ai/jobs](https://hud.ai/jobs). View results on your taskset's Leaderboard tab.

## After Your First Remote Run

Once you have results, the iteration cycle is fast:

| What changed | What to do |
|---|---|
| Task args or columns | `hud sync tasks` |
| Added/removed tasks | `hud sync tasks` |
| Grading logic or tools | `hud deploy` then re-run |
| Just want to re-run | `hud eval my-taskset claude --remote --full` |

## API Keys for Remote Runs

Remote runs use the HUD Gateway for model access. You have two options:

1. **BYOK (Bring Your Own Key):** Store your provider API keys at [hud.ai/project/api-keys](https://hud.ai/project/api-keys). Your keys are used at a lower credit cost.
2. **HUD Credits:** If no BYOK key is stored, pooled keys are used and charged against your credit balance.

Either way, you only need `HUD_API_KEY` — no provider-specific keys required for remote execution.

## Exporting Results

Pull your taskset data back to local files:

```bash
hud sync tasks my-taskset --export tasks.json
hud sync tasks my-taskset --export tasks.csv
```

CSV format flattens scenario args and custom columns into spreadsheet-friendly columns (`arg:user_id`, `col:difficulty`, etc.).
