<Update label="July 14, 2026">
## GPT-5.6 Support

- **GPT-5.6 family** ‚Äî `gpt-5.6` (Sol), `gpt-5.6-terra`, and `gpt-5.6-luna` are available through the
  HUD gateway and `create_agent`. `OpenAIAgent` now defaults to `gpt-5.6`.
- **Reasoning controls** ‚Äî GPT-5.6 reasoning passes through `OpenAIConfig.reasoning` unchanged: effort
  levels `none` through `xhigh` and `max`, and `mode` (`standard` or `pro`).
- **OpenAI SDK floor** ‚Äî the SDK now requires `openai>=2.45.0` (Responses API function-call outputs use
  the current content-item types).
</Update>

<Update label="May 6, 2026">
## Models, Tasksets, Templates & Sharing

### Platform

- **Models directory refresh** ‚Äî `/models` is a single unified list with Private and Trainable filters and a live usage column on every row.
- **Taskset analytics tab** ‚Äî dedicated analytics view on tasksets with charts and richer summaries.
- **Multi-environment taskset selection** ‚Äî pick multiple environments at once when configuring a taskset run.
- **Run from suggested tasksets** ‚Äî kick off an evaluation from a model's suggested-taskset row with the model already locked in.
- **Templates and workflow orchestration** ‚Äî templates settings page and a right-click workflow entry point for repeatable runs.
- **Resource sharing** ‚Äî invite users or whole teams to traces, jobs, evalsets, models, registry items, and collections with a unified accept flow.
- **Trace grader info** ‚Äî evaluation cards on traces show the grader that produced each result.
</Update>

<Update label="March 16, 2026">
## A2A Chat, Citations, GPT-5 & CLI Sync

- **A2A chat orchestrator** ‚Äî agent-to-agent communication for multi-agent workflows with input handling and follow-up turns.
- **`hud sync tasks`** ‚Äî sync task definitions from Python files or directories to the platform.
- **`hud sync env`** ‚Äî sync local environment configs with collision detection (replaces `hud link`).
- **`hud eval` accepts Python files** ‚Äî run evaluations directly from `.py` files and directories containing `Task` objects.
- **Chat class** ‚Äî manage multi-turn agent conversations from a single SDK abstraction.
- **GPT-5 support** ‚Äî `ResponseAgent` defaults to `gpt-5`, with ToolSearch tool support.
- **Citations** ‚Äî citation support for Claude, Gemini, and OpenAI responses in chat and agent traces.

### Platform

- **Click & scroll coordinate overlays** ‚Äî computer use traces render click coordinates and scroll actions directly on screenshots.
- **Trace-level QA workflows** ‚Äî run QA workflows across all tasks from the trace table, with screenshot input and per-task status.
- **Evalset environment filtering** ‚Äî filter results by environment version, with an earliest-version-only toggle.
- **EvaluationResult info viewer** ‚Äî inspect the full `info` field of evaluation results directly in the UI.
- **Individual user spend** ‚Äî usage page shows per-user spend alongside team totals.
- **Inline job renaming** ‚Äî rename jobs directly from the jobs page.
- **Modal integration** ‚Äî run environments on Modal compute infrastructure.
- **Resources section** ‚Äî new `/resources` page with published articles.
</Update>

<Update label="February 16, 2026">
## Opus 4.6 Computer Use, Streaming & Deploy Improvements

- **Opus 4.6 computer tool** ‚Äî native support for Claude Opus 4.6 computer use with zoom and screenshot gating.
- **Fine-grained tool streaming** ‚Äî opt-in streaming for individual tool results during agent execution.
- **`hud deploy` build args & secrets** ‚Äî pass build arguments and secrets to environment container builds.
- **`allowed_tools` in `@env.scenario`** ‚Äî scope tool access per evaluation scenario via the decorator.
- **Checkpoint configs** ‚Äî configure checkpoint behavior for long-running evaluations.

### Platform

- **Billing refactor** ‚Äî auto top-up, redesigned billing page, and per-key pricing for HUD-managed API keys.
- **Trace viewer enhancements** ‚Äî strip review mode, inline run switching, and file attachment display.
- **Trace comments** ‚Äî add and edit comments on individual traces, with a dedicated column in taskset view.
- **Training jobs dashboard** ‚Äî dedicated section for RL training jobs with detail pages.
- **Native binarization toggle** ‚Äî pass/fail binarization for taskset evaluations, built into the platform.
- **Column ordering** ‚Äî reorder columns in the taskset table view.
- **Model & environment sorting** ‚Äî sort taskset results by model, environment, and environment version.
</Update>

<Update label="January 12, 2026">
## CLI Refinements & Leaderboard Redesign

- **Build args for `hud deploy`** ‚Äî pass custom build arguments to environment container builds.
- **Wildcard tools** ‚Äî environments can expose `*` to allow all tools without explicit registration.
- **CLI mode distinction** ‚Äî `hud build` and `hud analyze` distinguish between HTTP and stdio modes.

### Platform

- **Leaderboard redesign** ‚Äî redesigned leaderboards with publishing flow, public visibility, and embedding support.
- **Slack bot** ‚Äî Slack integration for job notifications and external integration providers.
- **Trace compact view** ‚Äî compact trace view with column reorder, inline comments, and truncated task names.
- **BYOK API keys** ‚Äî bring-your-own-key support with a `use_hud_key` option for user-managed API keys.
- **Per-key pricing** ‚Äî individual pricing tiers for HUD-managed API keys.
- **Jobs page improvements** ‚Äî compact job list view and refreshed stats.
</Update>

<Update label="December 17, 2025">
## v0.5.0: MCP-First Architecture

- **Environments decoupled** ‚Äî environment definitions moved to separate repos, enabling independent versioning and community contributions.
- **Unified scenario/tool/prompt/resource handling** ‚Äî single abstraction layer for MCP servers and client-side tools, with caching and hot-reload.
- **Telemetry** ‚Äî trace IDs, subagent spans, and structured logging for agent runs.
- **Scenario decorator** ‚Äî `@env.scenario` for defining evaluation scenarios with typed configuration.
- **RL training** ‚Äî initial support for reinforcement learning training via the CLI.

### Platform

- **Inference API usage tracking** ‚Äî track inference API usage on the usage page.
- **HUD-managed API keys** ‚Äî platform-side API key management with `set api_key` support.
</Update>
